Quick Start

From zero to a running Oxide app in five steps.

1

Install Rust & WASM Toolchain

$ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
$ rustup target add wasm32-unknown-unknown
$ cargo install wasm-pack
2

Create Your Project

$ cargo new --lib my-app
$ cd my-app
3

Add Dependencies

Cargo.toml 
[lib]
crate-type = ["cdylib"]

[dependencies]
oxide = { git = "https://github.com/IEvangelist/Oxide" }
wasm-bindgen = "0.2"
4

Write Your App

src/lib.rs 
use oxide::prelude::*;
use wasm_bindgen::prelude::*;

#[wasm_bindgen(start)]
pub fn main() {
    mount("#app", || {
        let mut count = signal(0);
        view! {
            <button on:click={move |_| count += 1}>
                "Clicks: " {count}
            </button>
        }
    });
}
5

Build & Run

$ wasm-pack build --target web
$ python -m http.server 8080

CLI Reference

The oxide CLI handles project scaffolding, development, and production builds.

CommandDescription
oxide new <name>Scaffold a new Oxide project with default template, dependencies, and wasm32 target configured.
oxide devStart dev server with live reload, WASM compilation, and DWARF debug info for Chrome DevTools.
oxide buildProduction build with release optimizations, wasm-opt, and gzip-ready output.
oxide serveServe the built site from the dist/ directory on a local HTTP server.

Debugging Guide

Debug your Oxide app with full Rust source-level debugging in Chrome.

  1. Build with oxide dev — this enables debug mode and embeds DWARF debug info in the WASM binary.
  2. Open your app in Chrome and open DevTools (F12).
  3. Install the "C/C++ DevTools Support (DWARF)" Chrome extension.
  4. Go to Sources tab — you'll see your Rust source files listed.
  5. Set breakpoints directly in your .rs files. Step through code, inspect variables, view the call stack.
# Build with debug info
$ oxide dev

# Chrome will show Rust sources under:
# Sources → file:// → src/lib.rs
# Set breakpoints, step in/out, inspect locals

For production, oxide build strips all debug info and runs wasm-opt -Oz for the smallest possible binary.

OpenTelemetry Setup

Built-in observability. Traces are automatically emitted for signals, effects, and fetch calls.

src/lib.rs 
use oxide_telemetry;

// Initialize with console output (dev mode)
oxide_telemetry::init(Config {
    service_name: "my-app",
    endpoint: None, // console mode
});

// Or send to an OTLP collector
oxide_telemetry::init(Config {
    service_name: "my-app",
    endpoint: Some("http://localhost:4317"),
});

// All signals, effects, and fetches are now traced automatically.
// W3C trace context is propagated across fetch calls.

When no endpoint is configured, tracing has zero runtime overhead — the compiler eliminates all instrumentation code.

Resiliency Patterns

Production-grade fault tolerance built into the framework.

Error Boundary

view! {
    <ErrorBoundary fallback={|err| view! {
        <p class="error">"Something went wrong: " {err}</p>
    }}>
        <MyComponent />
    </ErrorBoundary>
}

Retry with Backoff

use oxide_resiliency::retry;

let data = retry(RetryConfig {
    max_attempts: 3,
    backoff: Backoff::Exponential { base_ms: 100 },
}, || async {
    fetch_json("https://api.example.com/data").await
}).await;

Circuit Breaker

use oxide_resiliency::circuit_breaker;

let breaker = circuit_breaker(CircuitConfig {
    failure_threshold: 5,
    reset_timeout_ms: 30_000,
});

let result = breaker.call(|| async {
    fetch_json("https://api.example.com/data").await
}).await;
